# W09 Web 小白验收打分表

用途：给 `Worker-09` 的前端结果做快速验收。

打分规则（每条三档）：
- 做到 = 5 分
- 一般 = 3 分
- 没做到 = 0 分

总分：20 条 × 5 分 = 100 分。

## A. 页面完整度（25 分）

1. 有项目列表页（能看到多个项目）
2. 点项目后能进入详情页
3. 详情页里能看到 job（运行记录）
4. 有日志区
5. 有进度和产物区（能看到文件/链接）

## B. 按钮与状态（25 分）

6. 有 `pause/resume/cancel/delete` 按钮
7. 点按钮后有“处理中”状态（如 loading）
8. 操作后有成功/失败反馈
9. 不会把 A 项目的操作发到 B 项目
10. 刷新页面后状态可恢复

## C. 实时与断线（25 分）

11. 页面进入先显示 snapshot（不是长时间空白）
12. 随后能看到 WS 增量更新（日志/进度）
13. WS 断开后会回退到 snapshot 轮询
14. WS 恢复后不重复刷屏、不乱序
15. WS 不稳定时，页面最终状态仍正确

## D. 视觉与可用性（25 分）

16. 不是默认脚手架风格（有明确配色与层次）
17. 字体可读、间距合理
18. 移动端 360px 宽度可用（不横向炸裂）
19. 关键区（日志/按钮/进度）容易找到
20. 有轻量动效，但不影响阅读和操作

## 结果判定

- 85-100：通过，可合并
- 70-84：可用，但需要补 2-4 项
- <70：退回重做

## 快速使用（10 分钟）

1. 打开页面，按 A->D 顺序逐条打分。
2. 记录每条得分（5/3/0）。
3. 计算总分。
4. 把 0 分项发回给 Worker-09，要求定点修复。
